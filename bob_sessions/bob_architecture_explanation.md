```python
async def get_iam_token():
    """Exchange IBM API key for IAM access token."""
    async with httpx.AsyncClient() as client:
        response = await client.post(
            "https://iam.cloud.ibm.com/identity/token",
            data={
                "grant_type": "urn:ibm:params:oauth:grant-type:apikey",
                "apikey": WATSONX_API_KEY  # From .env file
            },
            headers={"Content-Type": "application/x-www-form-urlencoded"}
        )
        return response.json().get("access_token", "")
```

**Why IAM Token?**
- IBM Cloud uses OAuth 2.0 for authentication
- API keys are exchanged for short-lived access tokens
- Tokens expire after ~1 hour (automatically refreshed per request)

```python
def generate_local_analysis(prompt: str, context: str = "", scores: dict = None) -> str:
    """Fallback analysis using context when watsonx is unavailable."""
    
    if "Architecture" in prompt or "analyze" in prompt.lower():
        # Parse the context string to extract key information
        lines = context.split('\n')
        lang = next((l.split(':')[1].strip() for l in lines if 'Primary Language:' in l), 'Multiple')
        has_tests = '✓' in next((l for l in lines if 'Has Tests:' in l), '')
        has_docker = '✓' in next((l for l in lines if 'Has Docker:' in l), '')
        has_ci = '✓' in next((l for l in lines if 'Has CI/CD:' in l), '')
        arch_type = next((l.split(':')[1].strip() for l in lines if 'Architecture:' in l), 'Unknown')
        
        # Calculate overall score from individual scores
        s = scores or {}
        avg = round(
            s.get('structure_score', 0) * 0.30 +
            s.get('test_score',      0) * 0.25 +
            s.get('docs_score',      0) * 0.20 +
            s.get('devops_score',    0) * 0.15 +
            s.get('security_score',  0) * 0.10
        , 1)
        
        # Generate structured analysis without AI
        return f"""## Architecture Summary
This is a **{arch_type}** application built primarily with **{lang}**.

## Key Components
- **Entry Points**: Core application files handling initialization and routing
- **Business Logic**: Domain-specific modules implementing core functionality
- **Configuration**: Environment settings and application configuration
- **Data Layer**: Models, schemas, and database interactions
- **Testing Suite**: {'Comprehensive test coverage' if has_tests else 'Tests not detected — consider adding them'}

## Tech Stack
- **Primary Language**: {lang}
- **Architecture**: {arch_type}
- **DevOps**: {'Docker + ' if has_docker else ''}{'CI/CD pipeline' if has_ci else 'No CI/CD detected'}

## Onboarding Guide
1. Review README and setup instructions
2. Check package files for dependencies
3. Explore main application files
4. {'Run the existing test suite' if has_tests else 'Set up your development environment'}

## Complexity Assessment
- **Overall Score**: {avg}/10 (computed from test coverage, DevOps, docs, structure, security)
- **Maintainability**: {'Good' if avg >= 6 else 'Needs improvement'}
- **Recommendation**: {'Well-structured project — focus on CI/CD and containerization.' if has_tests else 'Add tests and a CI/CD pipeline to improve reliability.'}"""
    else:
        # For general questions, provide generic response
        return "Based on the repository analysis, this codebase implements the requested functionality through well-organized modules. Review the key files in the architecture summary for implementation details."
```

**Cause**: Model timeout or invalid prompt

**Solution**: System automatically falls back to local analysis

### Issue: "Rate limit exceeded"

**Cause**: Too many requests to watsonx.ai

**Solution**: 
- Implement caching for repeated analyses
- Use fallback mechanism
- Upgrade IBM Cloud plan

### Issue: "Invalid project ID"

**Cause**: Wrong project ID in .env

**Solution**:
```bash
# Get project ID from watsonx.ai dashboard
# Update .env file
WATSONX_PROJECT_ID=correct_project_id_here
```

## 🔍 Detailed Analysis

### 1. Dockerfile Review

#### ❌ **Issues Found in Original Dockerfile**

```dockerfile
FROM python:3.11-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
EXPOSE 8000
CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]
```

**Problems:**
1. ❌ No `.dockerignore` file → Copies unnecessary files (tests, docs, .git)
2. ❌ Runs as root user → Security vulnerability
3. ❌ No health check → Code Engine can't monitor container health
4. ❌ Missing environment variables → Not optimized for Python
5. ❌ No system dependencies cleanup → Larger image size
6. ❌ Single worker → Not utilizing available CPU cores
7. ❌ No proper signal handling → Graceful shutdown issues

#### ✅ **Optimized Dockerfile** (Now Implemented)

**Key Improvements:**

1. **Security Enhancements**
   ```dockerfile
   # Create non-root user
   RUN useradd -m -u 1000 appuser
   USER appuser
   ```
   - Runs as non-root user (UID 1000)
   - Reduces attack surface
   - Follows security best practices

2. **Python Optimization**
   ```dockerfile
   ENV PYTHONUNBUFFERED=1 \
       PYTHONDONTWRITEBYTECODE=1 \
       PIP_NO_CACHE_DIR=1
   ```
   - `PYTHONUNBUFFERED=1` → Real-time logs in Code Engine
   - `PYTHONDONTWRITEBYTECODE=1` → No .pyc files (smaller image)
   - `PIP_NO_CACHE_DIR=1` → Faster builds, smaller layers

3. **Health Check**
   ```dockerfile
   HEALTHCHECK --interval=30s --timeout=10s --start-period=40s --retries=3 \
       CMD curl -f http://localhost:8000/health || exit 1
   ```
   - Automatic container health monitoring
   - Code Engine can restart unhealthy containers
   - 40s start period for slow startup

4. **Layer Caching**
   ```dockerfile
   COPY --chown=appuser:appuser requirements.txt .
   RUN pip install --no-cache-dir -r requirements.txt
   COPY --chown=appuser:appuser . .
   ```
   - Dependencies cached separately from code
   - Faster rebuilds when only code changes
   - Proper file ownership for non-root user

5. **System Cleanup**
   ```dockerfile
   RUN apt-get update && \
       apt-get install -y --no-install-recommends curl && \
       rm -rf /var/lib/apt/lists/*
   ```
   - Removes package lists after install
   - Reduces image size by ~50MB
